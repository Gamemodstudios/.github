# Contributing Guidelines

Thank you for your interest in contributing to our organization! We are thrilled to have you here. To maintain code quality and ensure a smooth review process, please follow these guidelines.

---

## 🛠️ Getting Started

1. **Fork the Repository:** Create a personal fork of the repository you wish to contribute to.
2. **Clone Locally:** Clone your fork to your local machine:
```bash
   git clone [https://github.com/YOUR-FORK-URL.git](https://github.com/YOUR-FORK-URL.git)
```

3. **Set up Upstream:** Keep your fork up to date by adding the main repository as an upstream remote:
```bash
git remote add upstream [https://github.com/Gamemodstudios/REPO-NAME.git](https://github.com/Gamemodstudios/REPO-NAME.git)

```



## 🌿 Branching Strategy

* Always create a new branch for your work. Do not commit directly to `main` or `master`.
* Use a descriptive branch naming convention:
* `feature/your-feature-name`
* `bugfix/issue-description`
* `docs/update-readme`



```bash
git checkout -b feature/short-description

```

## 📝 Commit Messages

We appreciate clear and concise commit messages. Whenever possible, use conventional commit styles:

* `feat: add user login validation`
* `fix: resolve crash on mobile safari`
* `docs: update setup instructions in contributing guide`

## 🚀 Submitting a Pull Request

1. **Sync your fork:** Before submitting, make sure your branch has the latest changes from the upstream `main` branch.
2. **Push your changes:** Push your feature branch to your GitHub fork.
3. **Open a PR:** Go to the original repository on GitHub and click "Compare & pull request".
4. **Follow the Template:** Fill out the PR template completely (this is the checklist we set up earlier!).
5. **Be Responsive:** Reviewers may request changes. Please check back on your PR to address feedback.

## 🤝 Code Review Process

All submissions require at least one maintainer review before merging. We look for:

* Code quality, readability, and adherence to existing project styles.
* Passing automated tests (CI/CD pipelines).
* Relevant documentation updates.

---

Thank you for helping us build great software!
